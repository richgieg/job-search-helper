# Sample Persona

## Overview

Rowan Mercer is the sample user persona for the repository's example data set. Rowan is an experienced technology professional using this application to maintain multiple tailored job-search profiles for closely related career paths.

The sample data should represent one real person adapting shared professional history into three base profiles:

1. Software Engineer
2. DevOps Engineer
3. Cloud Engineer

## Motivation

Rowan is actively exploring new roles and wants one place to manage tailored resumes, cover letters, links, experience bullets, education bullets, and job-specific application materials.

The purpose of the sample data is to demonstrate how the app supports:

- one shared identity across all profiles
- multiple reusable base profiles for adjacent role targets
- tailored positioning without inventing a completely different background for each role
- job tracking, interview tracking, application questions, and generated document previews

## Content Tone

The sample content should feel:

- realistic
- concise
- professional
- moderately polished
- specific enough to be believable without becoming overly detailed

Avoid:

- exaggerated claims
- overly generic AI-style phrasing
- unrealistic scope in a single bullet
- turning education bullets into inflated pseudo-work experience
- overly senior language that implies staff-plus leadership unless the work history supports it

## Naming Style

All people and company names used in the sample data should be fictional.

Guidance:

- use names that sound believable and professional, but not so common that they feel like real identifiable people
- prefer slightly stylized or less-common combinations over very ordinary first-name/last-name pairs
- avoid placeholder names such as John Doe, Jane Doe, Example Person, or anything joke-like
- avoid names that resemble public figures, celebrities, or well-known technology leaders
- keep the overall naming style consistent across the sample data so the data set feels intentionally fictional

The goal is for the sample data to read naturally while still signaling that it is fabricated example content.

## Sample Data Authoring Notes

When creating sample-data.json later:

- keep the three base profiles aligned to this persona
- reuse shared personal details exactly
- reuse the same underlying work history, education, certifications, and references wherever practical
- tailor summaries, cover letters, skill emphasis, experience bullets, and education bullets per profile
- make the sample data useful for demonstrating imports, exports, profile editing, job tracking, and document generation

## Sample Job Pipeline Guidance

The sample data set should include a realistic spread of job pursuits so the application's job-tracking workflows are easy to explore.

Guidance for sample-data.json:

- include about 15 jobs Rowan is pursuing or has recently pursued
- distribute those jobs across Software Engineer, DevOps Engineer, and Cloud Engineer targets
- include a mix of statuses, such as interested, applied, interview, withdrew, rejected, offer received, and offer accepted
- include some jobs with no interviews and some jobs with one or more interviews
- include some jobs with contacts, some with application questions, and some with both
- keep the companies fictional but believable
- make the job mix feel like one coherent search rather than 15 unrelated examples

This guidance should remain high-level. The actual list of jobs, contacts, interviews, and related application details belongs in sample-data.json, not in this persona document.
